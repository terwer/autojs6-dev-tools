## ADDED Requirements

### Requirement: 系统能够实时计算 OpenCV 模板匹配结果
系统 SHALL 使用 OpenCV TM_CCOEFF_NORMED 算法实时计算模板匹配，并返回匹配位置、置信度、耗时。

#### Scenario: 成功匹配并显示结果
- **WHEN** 用户加载模板图像并点击"匹配"按钮
- **THEN** 系统异步执行 OpenCV matchTemplate 计算
- **THEN** 画布绘制绿色矩形框标识匹配位置
- **THEN** 显示置信度数值（如 0.95）与耗时（如 120ms）

#### Scenario: 匹配置信度低于阈值
- **WHEN** 匹配结果置信度为 0.65，用户设置阈值为 0.80
- **THEN** 系统显示"未找到匹配"提示
- **THEN** 日志输出最高置信度值 0.65

#### Scenario: 多个匹配结果
- **WHEN** 模板在原图中存在多个匹配位置
- **THEN** 系统返回所有置信度高于阈值的匹配结果
- **THEN** 画布绘制所有匹配框，按置信度着色（绿色=高，黄色=中，橙色=低）

### Requirement: 系统能够支持阈值滑块实时调节
系统 SHALL 提供阈值滑块（范围 0.50~0.95），用户调节时实时更新匹配结果。

#### Scenario: 调节阈值更新匹配结果
- **WHEN** 用户将阈值从 0.80 调节至 0.70
- **THEN** 系统重新过滤匹配结果
- **THEN** 画布显示所有置信度 ≥ 0.70 的匹配框
- **THEN** 不重新执行 OpenCV 计算，仅过滤已有结果

#### Scenario: 阈值调节边界限制
- **WHEN** 用户尝试设置阈值低于 0.50
- **THEN** 系统限制最小阈值为 0.50
- **WHEN** 用户尝试设置阈值高于 0.95
- **THEN** 系统限制最大阈值为 0.95

### Requirement: 系统能够在画布上可视化匹配结果
系统 SHALL 在画布 Overlay 图层绘制匹配结果矩形框，并标注置信度数值。

#### Scenario: 绘制匹配框
- **WHEN** 匹配成功且置信度为 0.95
- **THEN** 画布绘制绿色矩形框（RGB: 16, 124, 16）
- **THEN** 矩形框位置与匹配结果坐标对齐

#### Scenario: 标注置信度数值
- **WHEN** 匹配框绘制完成
- **THEN** 矩形框右上角显示置信度数值（如 0.95）
- **THEN** 数值字体大小 12px，颜色白色，背景半透明黑色

#### Scenario: 多匹配结果着色
- **WHEN** 存在多个匹配结果
- **THEN** 置信度 ≥ 0.90 绘制绿色框
- **THEN** 置信度 0.80~0.89 绘制黄色框
- **THEN** 置信度 0.70~0.79 绘制橙色框

### Requirement: 系统能够输出匹配测试报告
系统 SHALL 输出匹配测试报告，包含耗时、坐标、状态、置信度详情。

#### Scenario: 输出成功匹配报告
- **WHEN** 匹配成功
- **THEN** 报告包含以下信息：
  - 状态：匹配成功
  - 耗时：120ms
  - 匹配位置：(100, 200)
  - 置信度：0.95
  - 匹配数量：1

#### Scenario: 输出失败匹配报告
- **WHEN** 匹配失败（置信度低于阈值）
- **THEN** 报告包含以下信息：
  - 状态：匹配失败
  - 耗时：120ms
  - 最高置信度：0.65
  - 阈值：0.80

### Requirement: 系统能够验证控件选择器有效性
系统 SHALL 验证生成的 UiSelector 选择器能否在当前 UI 树中找到对应控件。

#### Scenario: 选择器验证成功
- **WHEN** 用户点击"验证选择器"按钮
- **THEN** 系统在当前 UI 树中查找匹配节点
- **THEN** 找到匹配节点后高亮显示
- **THEN** 输出报告：验证成功，找到 1 个匹配节点

#### Scenario: 选择器验证失败
- **WHEN** 选择器在当前 UI 树中无匹配节点
- **THEN** 系统显示 Toast 提示"选择器无效，未找到匹配节点"
- **THEN** 输出报告：验证失败，建议检查 resource-id 或 text 属性

#### Scenario: 选择器匹配多个节点
- **WHEN** 选择器匹配到多个节点
- **THEN** 系统高亮显示所有匹配节点
- **THEN** 输出报告：验证成功，找到 3 个匹配节点（建议添加更精确的属性）

### Requirement: 系统能够对齐 Dump 坐标系与截图坐标系
系统 SHALL 确保 UI 树 bounds 坐标与截图像素坐标完全对齐，验证控件边界框位置准确性。

#### Scenario: 坐标系对齐验证
- **WHEN** 用户点击"验证坐标对齐"按钮
- **THEN** 系统对比控件边界框与截图中实际控件位置
- **THEN** 输出对齐误差（如 ±2 像素）

#### Scenario: 坐标系对齐失败
- **WHEN** 控件边界框与截图位置偏差超过 10 像素
- **THEN** 系统显示警告"坐标系对齐异常，请检查设备分辨率或 DPI 设置"
- **THEN** 日志输出详细偏差数据

### Requirement: 系统能够异步执行匹配测试避免 UI 阻塞
系统 SHALL 使用异步架构执行所有匹配测试操作（OpenCV 计算、选择器验证）。

#### Scenario: OpenCV 计算不阻塞 UI
- **WHEN** 系统执行模板匹配（耗时 50-200ms）
- **THEN** UI 保持响应，显示加载指示器
- **THEN** 计算完成后自动更新画布和报告

#### Scenario: 选择器验证不阻塞 UI
- **WHEN** 系统验证选择器（遍历 5000+ 节点）
- **THEN** UI 保持响应，显示加载指示器
- **THEN** 验证完成后自动高亮匹配节点

### Requirement: 系统能够支持批量模板匹配测试
系统 SHALL 支持加载多个模板图像，批量执行匹配测试并生成汇总报告。

#### Scenario: 批量匹配测试
- **WHEN** 用户加载 5 个模板图像并点击"批量匹配"
- **THEN** 系统依次执行每个模板的匹配测试
- **THEN** 输出汇总报告：成功 3 个，失败 2 个，总耗时 600ms

#### Scenario: 批量测试中断
- **WHEN** 用户在批量测试过程中点击"停止"按钮
- **THEN** 系统中断剩余测试
- **THEN** 输出已完成的测试结果
